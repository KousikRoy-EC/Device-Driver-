🔑 The Three Contexts — Never Mix Them Up

Every decision you make in a driver — which allocator flag, which lock type, whether you can sleep — depends on knowing which of the three contexts you're in:

  • Process context — running inside a syscall, ioctl, module init/exit, or kthread. Can sleep. current is valid. GFP_KERNEL is fine. Mutexes work.
  • Interrupt / Atomic context — inside an IRQ handler, softirq, tasklet, or while holding a spinlock. Cannot sleep — ever. Must use GFP_ATOMIC. Only spinlocks.
  • Workqueue / Deferred context — runs in a kworker kernel thread (process context). Can sleep. Use to defer work from interrupt context.

Knowledge Checks — Test Yourself

📝 12 Questions Covering All of Week 1

Read each question, pick your answer mentally, then check the highlighted correct answer and explanation. Be honest — if you miss more than 3, re-read that day's guide before moving to Week 2.

Q1 — You're in an IRQ handler and need to allocate 64 bytes for a packet header. Which call is correct?
kmalloc(64, GFP_KERNEL)
kmalloc(64, GFP_ATOMIC)
vmalloc(64)
kzalloc(64, GFP_KERNEL)
IRQ handlers are atomic context — sleeping is forbidden. GFP_KERNEL can sleep when memory is tight. GFP_ATOMIC never sleeps (uses emergency reserves). vmalloc() always sleeps internally. Always check the return value — GFP_ATOMIC may return NULL.
Q2 — Your driver's list_for_each_entry() loop needs to delete some entries that match a condition. What must you change?
Add a spinlock around the loop
Use list_del_rcu() inside the loop
Switch to list_for_each_entry_safe() with a tmp pointer
Nothing — list_del() inside a for_each loop is always safe
list_for_each_entry() saves only the current entry's next pointer at the top of each iteration. If you delete the current entry, that saved pointer is now invalid and the next iteration will dereference freed memory. list_for_each_entry_safe() saves next into a tmp variable before processing, making deletion safe.
Q3 — A kernel oops shows "RIP: hello_write+0x3c/0x90 [hello]". What does 0x3c/0x90 mean?
The function is at address 0x3c in kernel memory
The crash happened 90 bytes before the function end
The crash is 0x3c (60) bytes into the function; the function is 0x90 (144) bytes total
The function was called 0x3c times out of a maximum 0x90
The format is function+offset/size. Offset 0x3c from the start of hello_write is the exact instruction that faulted. To find the source line: addr2line -e hello.o 0x3c or use scripts/faddr2line hello.ko hello_write+0x3c.
Q4 — When should you use vmalloc() instead of kmalloc()?
When you need the fastest possible allocation
When the memory will be used for DMA transfers
When you need a large buffer (>128KB) and physical contiguity is not required
When you're in atomic context and GFP_ATOMIC is not available
vmalloc() allocates memory that is virtually contiguous but physically scattered. This lets it satisfy large allocations that kmalloc() would fail (fragmented physical memory). However: vmalloc is slower (TLB pressure), cannot be used for DMA (not physically contiguous), and always implies GFP_KERNEL (can sleep).
Q5 — What does container_of(ptr, struct sensor, list) actually compute?
The size of struct sensor
The base address of the containing struct sensor, by subtracting the offset of 'list' from ptr
A new pointer to a copy of the sensor struct
The field ptr points to, cast to struct sensor *
At compile time, offsetof(struct sensor, list) gives the byte distance from the struct's start to the list member. At runtime: (struct sensor *)((char *)ptr - offsetof(struct sensor, list)). GCC also type-checks that ptr matches the type of sensor.list, catching mistakes at compile time.
Q6 — Your kthread loop uses msleep(1000) inside while (!kthread_should_stop()). If kthread_stop() is called, what is the worst-case wait time before the thread exits?
Immediate — kthread_stop() forces the thread to exit
Up to 1 second — the thread wakes from msleep, checks kthread_should_stop(), then exits
Infinite — msleep() is uninterruptible
Depends on whether the thread holds a lock
kthread_stop() sets a flag and then waits for the thread to exit — it does not forcibly kill it. The thread must check kthread_should_stop() and return. With a 1-second sleep, the thread will check only once per second. To make threads more responsive, use msleep_interruptible(1000) — kthread_stop() sends a wakeup signal that cuts the sleep short.
Q7 — Which GFP flag prevents the allocator from starting any I/O during memory reclaim?
GFP_ATOMIC
GFP_NOWAIT
GFP_NOIO
GFP_DMA
GFP_NOIO prohibits the allocator from initiating I/O during memory reclaim. This is essential in block/storage drivers to prevent a deadlock where: driver needs memory → allocator tries to reclaim by flushing dirty pages → flush needs the driver → driver needs memory → deadlock. GFP_NOFS is similar but blocks filesystem operations specifically.
Q8 — You call wait_event_interruptible(wq, condition) and the user presses Ctrl+C. What does the function return?
0 — the wait succeeded
-EINTR
-ERESTARTSYS
1 — interrupted flag
wait_event_interruptible() returns -ERESTARTSYS when a signal arrives. Your driver's read/write/ioctl handler should check for this and return -EINTR to userspace (or -ERESTARTSYS to let the kernel automatically restart the syscall). The -ERESTARTSYS vs -EINTR distinction controls whether the C library retries the syscall transparently.
Q9 — A new developer uses BUG_ON() for every NULL pointer check in their driver. What's wrong with this?
BUG_ON() is not available in modules, only built-in drivers
Many NULL pointer conditions are recoverable — WARN_ON() with a graceful error return is safer and keeps the system alive for debugging
BUG_ON() has a significant performance overhead in hot paths
Nothing — BUG_ON(ptr == NULL) is the correct defensive programming style
Reserve BUG_ON() for true invariant violations where continuing execution would cause worse harm (data corruption, security breach). For unexpected-but-recoverable conditions, WARN_ON() prints the stack trace (invaluable for debugging) and lets the system continue running — giving you time to collect more information. A crashed system tells you nothing.
Q10 — What is the key structural difference between a process and a kernel thread from the kernel's perspective?
Kernel threads run in Ring 0; processes run in Ring 3
Kernel threads use a different struct than task_struct
Kernel threads have mm == NULL (no user address space); processes have a valid mm_struct
Kernel threads cannot be scheduled; they run to completion without preemption
Both processes and kernel threads are represented by task_struct and are scheduled identically. The key difference: kernel threads have task->mm == NULL — they have no user virtual address space. When a kernel thread runs, it borrows the active_mm of whatever process it displaced (for page table reasons) but never accesses user memory.
Q11 — Which kfifo characteristic makes it lock-free for the single-producer / single-consumer case?
It uses RCU (Read-Copy-Update) internally
The head (read) and tail (write) indices are modified by only one side each — no concurrent modification of the same variable
It disables interrupts around index updates
The kernel provides a hardware atomic instruction for ring buffer updates
The producer only ever writes to the tail index; the consumer only ever reads from the head index. Since each side modifies a different variable and they only read the other side's index, there is no conflicting concurrent write. Combined with memory barriers (which the kfifo implementation handles internally), this eliminates any need for a lock in the SPSC case.
Q12 — Your module has a work item scheduled via schedule_work(). In module exit, you call kfree(dev) and then cancel_work_sync(). What is wrong?
cancel_work_sync() should be called before module exit
The work function may still be running when kfree(dev) executes, causing a use-after-free — cancel_work_sync() must be called BEFORE kfree(dev)
schedule_work() and cancel_work_sync() cannot be used together
Nothing — kfree() is safe to call while a work item references dev
cancel_work_sync() cancels any pending work item and waits for any currently-running execution to finish. If you free dev first and the work function is still running (accessing dev fields), you have a use-after-free — one of the most dangerous kernel bugs. Correct order: cancel_work_sync() → then kfree(dev).

Mini Project — "Virtual Sensor Hub" Kernel Module

🏗️ What You're Building

The Virtual Sensor Hub is a self-contained kernel module that simulates a multi-sensor data acquisition system. It exercises every concept from Week 1 in a realistic driver pattern — not a toy hello-world, but the skeleton of a real device driver.

The module manages a collection of virtual sensors, each producing periodic readings. A background kthread polls them, stores readings in per-sensor kfifos, organises sensors in a linked list and hash table, and signals waiting readers via a wait queue.

/*
 * sensor_hub.c — Day 7 Week 1 Capstone: Virtual Sensor Hub
 *
 * Demonstrates all Week 1 concepts in one integrated module:
 *   - Module parameters, __init/__exit (Day 2)
 *   - pr_debug, WARN_ON, coding style (Day 3)
 *   - kzalloc/kfree, GFP_KERNEL (Day 4)
 *   - list_head, hashtable, IDR, kfifo (Day 5)
 *   - kthread, workqueue, wait_queue, completion (Day 6)
 *
 * Build:  make
 * Load:   sudo insmod sensor_hub.ko
 *         sudo insmod sensor_hub.ko num_sensors=5 poll_ms=200
 * Debug:  sudo sh -c 'echo "module sensor_hub +p" > \
 *             /sys/kernel/debug/dynamic_debug/control'
 */
#include <linux/init.h>
#include <linux/module.h>
#include <linux/kernel.h>
#include <linux/slab.h>         /* kzalloc, kfree */
#include <linux/list.h>         /* list_head */
#include <linux/hashtable.h>    /* DECLARE_HASHTABLE */
#include <linux/idr.h>          /* IDR */
#include <linux/kfifo.h>        /* kfifo */
#include <linux/kthread.h>      /* kthread_run/stop */
#include <linux/workqueue.h>    /* INIT_WORK, schedule_work */
#include <linux/wait.h>         /* wait_queue_head_t */
#include <linux/completion.h>   /* completion */
#include <linux/delay.h>        /* msleep_interruptible */
#include <linux/random.h>       /* get_random_u32 */
#include <linux/jiffies.h>      /* jiffies, time_after */
#include <linux/atomic.h>       /* atomic_t */

MODULE_LICENSE("GPL");
MODULE_AUTHOR("Your Name");
MODULE_DESCRIPTION("Week 1 Capstone: Virtual Sensor Hub");
MODULE_VERSION("1.0");

/* ── Module parameters ────────────────────────────────────────────── */
static int  num_sensors  = 3;
static int  poll_ms      = 500;
static int  max_readings = 16;     /* kfifo capacity per sensor */
static bool verbose      = false;

module_param(num_sensors,  int,  0444);
module_param(poll_ms,      int,  0444);
module_param(max_readings, int,  0444);
module_param(verbose,      bool, 0444);
MODULE_PARM_DESC(num_sensors,  "Number of virtual sensors (default: 3)");
MODULE_PARM_DESC(poll_ms,      "Poll interval in ms (default: 500)");
MODULE_PARM_DESC(max_readings, "kfifo depth per sensor (default: 16, must be power-of-2)");
MODULE_PARM_DESC(verbose,      "Verbose logging (default: false)");

/* ── Per-sensor statistics ────────────────────────────────────────── */
struct sensor_stats {
    int  min;
    int  max;
    long sum;
    int  count;
};

/* ── Virtual sensor — one per simulated device ────────────────────── */
struct virtual_sensor {
    int                id;          /* assigned by IDR */
    char               name[24];
    int                value;       /* current simulated reading */
    struct sensor_stats stats;

    struct list_head   list;        /* sensor_hub.sensor_list */
    struct hlist_node  hnode;       /* sensor_hub.sensor_htable */

    struct kfifo       readings;    /* dynamic kfifo (set at init) */
    bool               data_ready;
};

/* ── Hub: global driver state ─────────────────────────────────────── */
struct sensor_hub {
    struct task_struct   *poll_thread;
    struct work_struct    process_work;
    wait_queue_head_t     data_wq;
    struct completion     first_reading;

    struct list_head      sensor_list;
    DECLARE_HASHTABLE(sensor_htable, 4); /* 16 buckets */
    struct idr            sensor_idr;
    int                   sensor_count;

    atomic_t              poll_count;
    bool                  any_data_ready;
};

static struct sensor_hub hub;

/* ── Simulate a sensor reading (replaces real hardware read) ──────── */
static int simulate_reading(struct virtual_sensor *s)
{
    /* Walk from previous value ±10, clamped to 0-100 */
    int delta = (int)(get_random_u32() % 21) - 10;
    s->value = clamp(s->value + delta, 0, 100);
    return s->value;
}

/* ── Update per-sensor statistics ─────────────────────────────────── */
static void update_stats(struct sensor_stats *st, int val)
{
    if (st->count == 0) {
        st->min = st->max = val;
    } else {
        st->min = min(st->min, val);
        st->max = max(st->max, val);
    }
    st->sum += val;
    st->count++;
}

/* ── Work handler: drains fifos, computes stats ───────────────────── */
static void hub_process_work_fn(struct work_struct *work)
{
    struct virtual_sensor *s;
    int val, processed = 0;

    pr_debug("hub: process_work running in '%s'\n", current->comm);

    list_for_each_entry(s, &hub.sensor_list, list) {
        while (kfifo_get(&s->readings, &val)) {
            update_stats(&s->stats, val);
            processed++;

            if (verbose)
                pr_info("  sensor[%d] %s: reading=%d (min=%d max=%d avg=%ld)\n",
                        s->id, s->name, val,
                        s->stats.min, s->stats.max,
                        s->stats.count ? s->stats.sum / s->stats.count : 0);
        }
    }

    if (processed > 0) {
        hub.any_data_ready = true;
        complete_all(&hub.first_reading);  /* harmless after first time */
        wake_up_interruptible(&hub.data_wq);
        pr_debug("hub: processed %d readings\n", processed);
    }
}

/* ── Poll thread: periodic sensor sampling ────────────────────────── */
static int hub_poll_thread(void *data)
{
    struct virtual_sensor *s;

    pr_info("hub: poll thread started (pid=%d, interval=%dms)\n",
            current->pid, poll_ms);

    while (!kthread_should_stop()) {
        int poll_num = atomic_inc_return(&hub.poll_count);

        pr_debug("hub: poll #%d\n", poll_num);

        list_for_each_entry(s, &hub.sensor_list, list) {
            int reading = simulate_reading(s);

            /* Queue reading into per-sensor kfifo */
            if (!kfifo_put(&s->readings, reading)) {
                pr_warn_ratelimited("hub: sensor[%d] fifo full, dropping reading\n",
                                    s->id);
            }
        }

        /* Defer processing to workqueue (process context) */
        schedule_work(&hub.process_work);

        /* Sleep, but wake immediately if kthread_stop() is called */
        msleep_interruptible(poll_ms);
    }

    pr_info("hub: poll thread stopping after %d polls\n",
            atomic_read(&hub.poll_count));
    return 0;
}

/* ── Sensor lifecycle helpers ─────────────────────────────────────── */
static struct virtual_sensor *sensor_create(int idx)
{
    struct virtual_sensor *s;
    int id, ret;

    s = kzalloc(sizeof(*s), GFP_KERNEL);
    if (!s)
        return NULL;

    /* Allocate unique ID from IDR */
    id = idr_alloc(&hub.sensor_idr, s, 0, 256, GFP_KERNEL);
    if (id < 0) {
        kfree(s);
        return NULL;
    }

    s->id    = id;
    s->value = 50;   /* start at midpoint */
    snprintf(s->name, sizeof(s->name), "sensor-%d", id);

    /* Allocate kfifo — size must be power of 2 */
    ret = kfifo_alloc(&s->readings, roundup_pow_of_two(max_readings),
                      GFP_KERNEL);
    if (ret) {
        idr_remove(&hub.sensor_idr, id);
        kfree(s);
        return NULL;
    }

    /* Add to linked list (insertion order) */
    list_add_tail(&s->list, &hub.sensor_list);

    /* Add to hash table (keyed by id) */
    hash_add(hub.sensor_htable, &s->hnode, hash_32(id, 4));

    hub.sensor_count++;
    pr_info("hub: created %s (id=%d, fifo_depth=%u)\n",
            s->name, id, kfifo_size(&s->readings));
    return s;
}

static void sensor_destroy(struct virtual_sensor *s)
{
    pr_info("hub: destroying %s (polls=%d, min=%d, max=%d, avg=%ld)\n",
            s->name, s->stats.count,
            s->stats.min, s->stats.max,
            s->stats.count ? s->stats.sum / s->stats.count : 0);

    hash_del(&s->hnode);
    list_del(&s->list);
    idr_remove(&hub.sensor_idr, s->id);
    kfifo_free(&s->readings);
    kfree(s);
    hub.sensor_count--;
}

/* ── Find sensor by ID using hash table ───────────────────────────── */
static struct virtual_sensor *sensor_find_by_id(int id)
{
    struct virtual_sensor *s;

    hash_for_each_possible(hub.sensor_htable, s, hnode, hash_32(id, 4)) {
        if (s->id == id)
            return s;
    }
    return NULL;
}

/* ── Module init ──────────────────────────────────────────────────── */
static int __init sensor_hub_init(void)
{
    int i, ret;
    struct virtual_sensor *s;

    pr_info("hub: initialising (sensors=%d, poll=%dms, fifo=%d)\n",
            num_sensors, poll_ms, max_readings);

    /* Validate parameters */
    if (WARN_ON(num_sensors < 1 || num_sensors > 32)) {
        pr_err("hub: num_sensors must be 1-32\n");
        return -EINVAL;
    }
    if (WARN_ON(poll_ms < 10 || poll_ms > 60000)) {
        pr_err("hub: poll_ms must be 10-60000\n");
        return -EINVAL;
    }

    /* Initialise hub state */
    INIT_LIST_HEAD(&hub.sensor_list);
    hash_init(hub.sensor_htable);
    idr_init(&hub.sensor_idr);
    init_waitqueue_head(&hub.data_wq);
    init_completion(&hub.first_reading);
    INIT_WORK(&hub.process_work, hub_process_work_fn);
    atomic_set(&hub.poll_count, 0);

    /* Create sensors — goto cleanup on any failure */
    for (i = 0; i < num_sensors; i++) {
        s = sensor_create(i);
        if (!s) {
            pr_err("hub: failed to create sensor %d\n", i);
            ret = -ENOMEM;
            goto err_sensors;
        }
    }

    /* Start the poll thread */
    hub.poll_thread = kthread_run(hub_poll_thread, NULL, "sensor_hub_poll");
    if (IS_ERR(hub.poll_thread)) {
        ret = PTR_ERR(hub.poll_thread);
        hub.poll_thread = NULL;
        pr_err("hub: failed to start poll thread: %d\n", ret);
        goto err_sensors;
    }

    /* Wait for at least one poll cycle to complete */
    pr_info("hub: waiting for first readings...\n");
    if (!wait_for_completion_timeout(&hub.first_reading,
                                    msecs_to_jiffies(poll_ms * 3))) {
        pr_warn("hub: timeout waiting for first readings\n");
    }

    pr_info("hub: ready with %d sensors\n", hub.sensor_count);
    return 0;

err_sensors: {
    struct virtual_sensor *tmp;
    list_for_each_entry_safe(s, tmp, &hub.sensor_list, list)
        sensor_destroy(s);
    idr_destroy(&hub.sensor_idr);
    return ret;
    }
}

/* ── Module exit ──────────────────────────────────────────────────── */
static void __exit sensor_hub_exit(void)
{
    struct virtual_sensor *s, *tmp;

    pr_info("hub: shutting down after %d poll cycles\n",
            atomic_read(&hub.poll_count));

    /* 1. Stop polling — no more new data after this */
    if (hub.poll_thread)
        kthread_stop(hub.poll_thread);

    /* 2. Wait for any in-flight work to complete */
    cancel_work_sync(&hub.process_work);

    /* 3. Wake any sleeping readers so they can exit */
    wake_up_all(&hub.data_wq);

    /* 4. Destroy all sensors (safe iteration) */
    list_for_each_entry_safe(s, tmp, &hub.sensor_list, list)
        sensor_destroy(s);

    /* 5. Clean up IDR */
    idr_destroy(&hub.sensor_idr);

    pr_info("hub: shutdown complete\n");
}

module_init(sensor_hub_init);
module_exit(sensor_hub_exit);
