Overview Why virtual filesystems matter for drivers

The Linux kernel provides three pseudo-filesystems that let drivers expose information and accept configuration from userspace without using custom ioctl calls or reading raw device files. Each serves a distinct purpose:

/proc
procfs
→
/sys
sysfs

/sys/kernel/debug

→
userspace

💡
Mental Model

procfs = runtime diagnostics & process info  |  sysfs = stable device attributes the user can read/write  |  debugfs = internal debug dumps, never ship in production

📋

procfs /proc — the original kernel window

procfs is one of the oldest kernel interfaces. It was originally designed to expose process information (hence the name) but drivers have long used it to publish runtime statistics. It lives at /proc.

Core API Functions

proc_create()
proc_create(name, mode, parent, fops)
Create a /proc file with given name, permissions, parent directory, and file_operations. Returns struct proc_dir_entry* or NULL on failure.
proc_mkdir()
proc_mkdir(name, parent)
Create a /proc directory. Pass NULL as parent to create directly under /proc. Returns a proc_dir_entry you use as parent for children.
proc_remove()
proc_remove(pde)
Recursively removes the proc entry and all children. Call this in your module_exit(). Replaces the old remove_proc_entry().
struct proc_ops
proc_ops { .proc_open, .proc_read, .proc_write, .proc_release }
Modern replacement for file_operations when used with /proc (since 5.6+). Use this, not the old file_operations struct.
⚠️
Kernel 5.6+ Change

The proc_ops struct replaced file_operations in procfs since Linux 5.6. If you see old code using file_operations with proc_create, it still works but the new struct proc_ops is preferred.

Minimal procfs Entry Example

mydriver_proc.c C 
#include <linux/module.h>
#include <linux/proc_fs.h>
#include <linux/uaccess.h>
#include <linux/seq_file.h>

static struct proc_dir_entry *proc_dir, *proc_file;
static int  read_count = 0;
static int  write_count = 0;

/* Called when userspace opens /proc/mydriver/stats */
static int proc_show(struct seq_file *m, void *v)
{
    seq_printf(m, "reads  : %d\n", read_count);
    seq_printf(m, "writes : %d\n", write_count);
    return 0;
}

static int proc_open(struct inode *inode, struct file *file)
{
    return single_open(file, proc_show, NULL);
}

static const struct proc_ops my_proc_ops = {
    .proc_open    = proc_open,
    .proc_read    = seq_read,      /* provided by seq_file */
    .proc_lseek   = seq_lseek,
    .proc_release = single_release,
};

static int __init mydrv_init(void)
{
    /* Create /proc/mydriver/ directory */
    proc_dir = proc_mkdir("mydriver", NULL);
    if (!proc_dir)
        return -ENOMEM;

    /* Create /proc/mydriver/stats file */
    proc_file = proc_create("stats", 0444, proc_dir, &my_proc_ops);
    if (!proc_file) {
        proc_remove(proc_dir);
        return -ENOMEM;
    }

    pr_info("mydriver: /proc/mydriver/stats created\n");
    return 0;
}

static void __exit mydrv_exit(void)
{
    proc_remove(proc_dir);  /* removes dir AND all children */
    pr_info("mydriver: proc entries removed\n");
}

seq_file API The correct way to write /proc files

Never write a /proc file using raw read() implementations. The seq_file interface handles the critical problem of partial reads — when userspace's buffer is smaller than your output. Raw read implementations frequently break with large output or when cat makes multiple read() calls.

simple_seq.c C — single_open pattern 
/* Use single_open when your entire output fits in one show() call.
   Most driver statistics files use this pattern. */

static int my_show(struct seq_file *m, void *v)
{
    seq_printf(m, "irq_count   : %lu\n", drv.irq_count);
    seq_printf(m, "bytes_rx    : %lu\n", drv.bytes_rx);
    seq_printf(m, "bytes_tx    : %lu\n", drv.bytes_tx);
    seq_printf(m, "error_count : %lu\n", drv.errors);
    return 0;  /* 0 = success */
}

static int my_open(struct inode *inode, struct file *file)
{
    return single_open(file, my_show, NULL);
    /* 3rd arg is "private data" passed as v to show() */
}

static const struct proc_ops fops = {
    .proc_open    = my_open,
    .proc_read    = seq_read,
    .proc_lseek   = seq_lseek,
    .proc_release = single_release,
};
iter_seq.c C — seq_operations iterator 
/* Use seq_operations when iterating over a list/array —
   e.g., printing all entries in a linked list or table. */

static void *my_seq_start(struct seq_file *s, loff_t *pos)
{
    if (*pos >= drv.entry_count) return NULL; /* EOF */
    return &drv.entries[*pos];  /* return pointer to current item */
}

static void *my_seq_next(struct seq_file *s, void *v, loff_t *pos)
{
    (*pos)++;
    if (*pos >= drv.entry_count) return NULL;
    return &drv.entries[*pos];
}

static void my_seq_stop(struct seq_file *s, void *v) { /* cleanup if needed */ }

static int my_seq_show(struct seq_file *s, void *v)
{
    struct my_entry *e = v;
    seq_printf(s, "%s : %d\n", e->name, e->value);
    return 0;
}

static const struct seq_operations my_seq_ops = {
    .start = my_seq_start,
    .next  = my_seq_next,
    .stop  = my_seq_stop,
    .show  = my_seq_show,
};

static int my_open(struct inode *inode, struct file *file)
{
    return seq_open(file, &my_seq_ops);
}

Key seq_file Output Functions

seq_printf()
seq_printf(m, fmt, ...)
Printf-style output into the seq buffer. Does NOT truncate — the buffer grows automatically. Most common function you'll use.
seq_puts() / seq_putc()
seq_puts(m, str), seq_putc(m, ch)
Write a string or single char. Faster than seq_printf when no formatting needed.
seq_write()
seq_write(m, buf, size)
Write raw binary data. Useful for hex dumps or binary-format debug output.
seq_hex_dump()
seq_hex_dump(m, prefix, style, rowsize, groupsize, buf, len, ascii)
Formatted hex dump — ideal for register dumps and buffer inspection.

sysfs /sys — the modern, structured interface

sysfs is the preferred interface for exporting device attributes. Unlike procfs (which is freeform), sysfs enforces a one-value-per-file rule. This makes it scriptable, discoverable by udev, and the official ABI for driver configuration.

📌
The One-Value Rule

Each sysfs file should contain exactly one value (a number, string, or on/off). This is a strict kernel ABI rule. If you need to expose multiple values, use multiple sysfs files or use debugfs.

Creating Sysfs Attributes — Three Approaches

sysfs_device_attr.c C 
#include <linux/device.h>

/* ── SHOW (read): called when user does: cat /sys/class/mydev/mydev0/threshold ── */
static ssize_t threshold_show(struct device *dev,
                               struct device_attribute *attr,
                               char *buf)
{
    struct my_device *mydev = dev_get_drvdata(dev);
    return sysfs_emit(buf, "%d\n", mydev->threshold);
    /* sysfs_emit() is the modern, safe replacement for sprintf(buf,...) */
}

/* ── STORE (write): called when user does: echo 50 > /sys/.../threshold ── */
static ssize_t threshold_store(struct device *dev,
                                struct device_attribute *attr,
                                const char *buf, size_t count)
{
    struct my_device *mydev = dev_get_drvdata(dev);
    int val;

    if (kstrtoint(buf, 10, &val))  /* safe string-to-int conversion */
        return -EINVAL;

    if (val < 0 || val > 100)
        return -EINVAL;  /* validate before applying */

    mydev->threshold = val;
    return count;  /* MUST return count on success, not 0 */
}

/* ── Macro to declare the attribute ── */
static DEVICE_ATTR_RW(threshold);  /* creates dev_attr_threshold, mode 0644 */
static DEVICE_ATTR_RO(status);     /* read-only,  mode 0444 */

/* ── Group attributes together (cleaner than adding one by one) ── */
static struct attribute *mydev_attrs[] = {
    &dev_attr_threshold.attr,
    &dev_attr_status.attr,
    NULL,  /* MUST be NULL-terminated */
};
ATTRIBUTE_GROUPS(mydev);  /* creates mydev_groups[] */

/* ── In device creation: pass the group ── */
static struct device *dev = device_create_with_groups(
    cls, NULL, devnum, mydev_ptr, mydev_groups, "mydev%d", minor);
sysfs_class_attr.c C — class_attribute 
/* Class attributes appear under /sys/class/myclass/
   and are shared across all instances of the class. */

static ssize_t version_show(struct class *cls,
                             struct class_attribute *attr,
                             char *buf)
{
    return sysfs_emit(buf, "1.2.3\n");
}

static CLASS_ATTR_RO(version);  /* creates class_attr_version */

static struct attribute *myclass_attrs[] = {
    &class_attr_version.attr,
    NULL
};
ATTRIBUTE_GROUPS(myclass);

struct class my_class = {
    .name           = "myclass",
    .class_groups   = myclass_groups,
};
sysfs_kobject.c C — raw kobject (advanced)
/* Use kobject_create_and_add() when you want a sysfs dir that isn't tied to a device or class. */ static struct kobject *my_kobj; static ssize_t foo_show(struct kobject *kobj, struct kobj_attribute *attr, char *buf) { return sysfs_emit(buf, "%d\n", my_val); } static struct kobj_attribute foo_attr = __ATTR_RO(foo); static int __init init(void) { /* Creates /sys/kernel/mydriver/ */ my_kobj = kobject_create_and_add("mydriver", kernel_kobj); if (!my_kobj) return -ENOMEM; return sysfs_create_file(my_kobj, &foo_attr.attr); } static void __exit cleanup(void) { kobject_put(my_kobj); /* decrements refcount, frees when 0 */ }
🚫
Common Mistakes in sysfs store()

1) Using sscanf() instead of kstrtoint() — sscanf doesn't detect overflow.
2) Returning 0 on success instead of count — causes infinite write loops in userspace.
3) Not validating input range — never trust userspace data.

Important Macros Reference

attribute_macros.c C — reference 
/* ── Device Attribute Macros ── */
DEVICE_ATTR_RO(name);          /* mode 0444, requires name_show() */
DEVICE_ATTR_WO(name);          /* mode 0200, requires name_store() */
DEVICE_ATTR_RW(name);          /* mode 0644, requires both */
DEVICE_ATTR(name, mode, show, store); /* full control */

/* ── kobj_attribute Macros ── */
__ATTR_RO(name);               /* for kobject-based attrs */
__ATTR_RW(name);
__ATTR(name, mode, show, store);

/* ── Safe string conversions (always use these in store()) ── */
kstrtoint(buf, base, &result);   /* string → int */
kstrtoul(buf, base, &result);    /* string → unsigned long */
kstrtobool(buf, &result);        /* "1","y","on","true" → true */
sysfs_emit(buf, fmt, ...);       /* modern show() output (replaces sprintf) */

kobject_lifecycle.c C 
/* kobject lifecycle — reference counting matters */

/* Create a kobject under /sys/kernel/mydir/ */
kobj = kobject_create_and_add("mydir", kernel_kobj);

/* kobjects use reference counting — never free directly */
kobject_get(kobj);     /* increment refcount */
kobject_put(kobj);     /* decrement; frees when count hits 0 */

/* Add/remove individual attribute files */
sysfs_create_file(kobj, &my_attr.attr);
sysfs_remove_file(kobj, &my_attr.attr);

/* Create a symlink in sysfs (used by driver model) */
sysfs_create_link(kobj, target_kobj, "link_name");
sysfs_remove_link(kobj, "link_name");

/* Send a uevent (triggers udev rules) */
kobject_uevent(kobj, KOBJ_ADD);    /* triggers udev SUBSYSTEM add */
kobject_uevent(kobj, KOBJ_REMOVE); /* triggers udev SUBSYSTEM remove */
kobject_uevent(kobj, KOBJ_CHANGE); /* triggers udev CHANGE event */

debugfs /sys/kernel/debug — your private lab

debugfs is an informal filesystem with no ABI requirements. You can dump anything here — full register maps, internal state machines, performance histograms. It is mounted at /sys/kernel/debug and is available only when CONFIG_DEBUG_FS=y.

🔒
Never Ship debugfs in Production

debugfs files expose raw internals. They can be read by root and may leak security-sensitive info (register values, internal pointers). Use #ifdef DEBUG guards or ensure you remove debugfs entries in release builds.

debugfs API

debugfs_create_dir()
debugfs_create_dir(name, parent)
Create a directory. Pass NULL as parent to create directly under /sys/kernel/debug. Returns a dentry pointer — never NULL (on failure returns error pointer).
debugfs_create_file()
debugfs_create_file(name, mode, parent, data, fops)
Create a file with full control via file_operations. Use for complex outputs like register dumps.
debugfs_create_u32()
debugfs_create_u32(name, mode, parent, &val)
Create a file that directly exposes a u32 variable. Kernel handles the read/write. Variants: u8, u16, u64, ulong, bool, atomic_t.
debugfs_remove_recursive()
debugfs_remove_recursive(dentry)
Remove a directory and all its contents. Call in module_exit() with your root dentry.
debugfs_example.c C 
#include <linux/debugfs.h>

static struct dentry *dbg_root;

/* Custom file: full register dump */
static int regdump_show(struct seq_file *m, void *v)
{
    seq_printf(m, "REG_CTRL   = 0x%08x\n", readl(drv.base + 0x00));
    seq_printf(m, "REG_STATUS = 0x%08x\n", readl(drv.base + 0x04));
    seq_printf(m, "REG_IRQ    = 0x%08x\n", readl(drv.base + 0x08));
    return 0;
}
DEFINE_SHOW_ATTRIBUTE(regdump);  /* creates regdump_fops automatically */

static int __init mydrv_init(void)
{
    /* Create /sys/kernel/debug/mydriver/ */
    dbg_root = debugfs_create_dir("mydriver", NULL);

    /* ── Simple scalar files — one line each ── */
    debugfs_create_u32("irq_count", 0444, dbg_root, &drv.irq_count);
    debugfs_create_u32("error_count", 0444, dbg_root, &drv.errors);
    debugfs_create_bool("debug_mode", 0644, dbg_root, &drv.debug);

    /* ── Complex file (register dump) ── */
    debugfs_create_file("registers", 0400, dbg_root, NULL, &regdump_fops);

    return 0;
}

static void __exit mydrv_exit(void)
{
    debugfs_remove_recursive(dbg_root);  /* removes dir + all files */
}
⚡
DEFINE_SHOW_ATTRIBUTE Macro

The DEFINE_SHOW_ATTRIBUTE(name) macro auto-generates name_fops with proper single_open, seq_read, llseek and release — saves ~10 lines of boilerplate. Use it whenever you just need a read-only show function.

Full Example Character driver with all three interfaces

This is a complete, compilable driver that exposes the same circular buffer from Day 9/10 through all three virtual filesystems. This is the Day 11 practice goal.

virtual_sensor.c C — complete driver 
#include <linux/module.h>
#include <linux/kernel.h>
#include <linux/fs.h>
#include <linux/cdev.h>
#include <linux/device.h>
#include <linux/proc_fs.h>
#include <linux/seq_file.h>
#include <linux/debugfs.h>
#include <linux/uaccess.h>
#include <linux/mutex.h>

#define DRIVER_NAME  "vsensor"
#define BUF_SIZE     1024

struct vsensor_dev {
    struct cdev    cdev;
    struct device *device;
    struct mutex   lock;
    char           buf[BUF_SIZE];
    size_t         data_len;
    unsigned long  reads, writes;     /* stats */
    int            threshold;          /* sysfs-configurable */
};

static struct vsensor_dev vdev;
static struct class       *vsensor_class;
static struct proc_dir_entry *proc_dir;
static struct dentry       *dbg_root;
static dev_t               devnum;

/* ════════════════════ procfs ════════════════════ */

static int proc_stats_show(struct seq_file *m, void *v)
{
    seq_printf(m, "driver     : %s\n", DRIVER_NAME);
    seq_printf(m, "reads      : %lu\n", vdev.reads);
    seq_printf(m, "writes     : %lu\n", vdev.writes);
    seq_printf(m, "buf_used   : %zu / %d bytes\n", vdev.data_len, BUF_SIZE);
    seq_printf(m, "threshold  : %d\n", vdev.threshold);
    return 0;
}
static int proc_stats_open(struct inode *i, struct file *f)
{
    return single_open(f, proc_stats_show, NULL);
}
static const struct proc_ops proc_stats_ops = {
    .proc_open    = proc_stats_open,
    .proc_read    = seq_read,
    .proc_lseek   = seq_lseek,
    .proc_release = single_release,
};

/* ════════════════════ sysfs ════════════════════ */

static ssize_t threshold_show(struct device *dev,
                               struct device_attribute *attr, char *buf)
{
    return sysfs_emit(buf, "%d\n", vdev.threshold);
}

static ssize_t threshold_store(struct device *dev,
    struct device_attribute *attr, const char *buf, size_t count)
{
    int val;
    if (kstrtoint(buf, 10, &val) || val < 0 || val > 1000)
        return -EINVAL;
    mutex_lock(&vdev.lock);
    vdev.threshold = val;
    mutex_unlock(&vdev.lock);
    return count;
}
static DEVICE_ATTR_RW(threshold);

static ssize_t stats_show(struct device *dev,
                            struct device_attribute *attr, char *buf)
{
    return sysfs_emit(buf, "reads=%lu writes=%lu\n",
                      vdev.reads, vdev.writes);
}
static DEVICE_ATTR_RO(stats);

static struct attribute *vsensor_attrs[] = {
    &dev_attr_threshold.attr,
    &dev_attr_stats.attr,
    NULL
};
ATTRIBUTE_GROUPS(vsensor);

/* ════════════════════ debugfs ════════════════════ */

static int buf_dump_show(struct seq_file *m, void *v)
{
    seq_printf(m, "data_len: %zu\n", vdev.data_len);
    seq_hex_dump(m, "buf: ", DUMP_PREFIX_OFFSET,
                 16, 1, vdev.buf, vdev.data_len, true);
    return 0;
}
DEFINE_SHOW_ATTRIBUTE(buf_dump);  /* generates buf_dump_fops */

/* ════════════════════ module init/exit ════════════════════ */

static int __init vsensor_init(void)
{
    int ret;

    mutex_init(&vdev.lock);
    vdev.threshold = 50;

    /* char device setup (from Day 8) */
    ret = alloc_chrdev_region(&devnum, 0, 1, DRIVER_NAME);
    if (ret) return ret;

    vsensor_class = class_create(DRIVER_NAME);
    if (IS_ERR(vsensor_class)) { ret = PTR_ERR(vsensor_class); goto err_unreg; }

    cdev_init(&vdev.cdev, &vsensor_fops);
    ret = cdev_add(&vdev.cdev, devnum, 1);
    if (ret) goto err_class;

    vdev.device = device_create_with_groups(vsensor_class, NULL,
                    devnum, &vdev, vsensor_groups, DRIVER_NAME);
    if (IS_ERR(vdev.device)) { ret = PTR_ERR(vdev.device); goto err_cdev; }

    /* ── procfs ── */
    proc_dir = proc_mkdir(DRIVER_NAME, NULL);
    proc_create("stats", 0444, proc_dir, &proc_stats_ops);

    /* ── debugfs ── */
    dbg_root = debugfs_create_dir(DRIVER_NAME, NULL);
    debugfs_create_file("buf_dump", 0400, dbg_root, NULL, &buf_dump_fops);
    debugfs_create_u32("read_count", 0444, dbg_root,
                       (u32*)&vdev.reads);

    pr_info("%s: loaded. /dev/%s, /proc/%s/, /sys/kernel/debug/%s/\n",
           DRIVER_NAME, DRIVER_NAME, DRIVER_NAME, DRIVER_NAME);
    return 0;

err_cdev:   cdev_del(&vdev.cdev);
err_class:  class_destroy(vsensor_class);
err_unreg:  unregister_chrdev_region(devnum, 1);
    return ret;
}

static void __exit vsensor_exit(void)
{
    debugfs_remove_recursive(dbg_root);
    proc_remove(proc_dir);
    device_destroy(vsensor_class, devnum);
    cdev_del(&vdev.cdev);
    class_destroy(vsensor_class);
    unregister_chrdev_region(devnum, 1);
    pr_info("%s: unloaded\n", DRIVER_NAME);
}

module_init(vsensor_init);
module_exit(vsensor_exit);
MODULE_LICENSE("GPL");
MODULE_AUTHOR("Your Name");
MODULE_DESCRIPTION("Virtual sensor driver — procfs/sysfs/debugfs demo");

Best Practices What separates good driver code from great

🔢
sysfs_emit() over sprintf()
Always use sysfs_emit() in show() callbacks. It bounds-checks against PAGE_SIZE and prevents buffer overflows that sprintf(buf, ...) ignores.
↩️
Return count in store()
sysfs store() MUST return count on success, not 0. Returning 0 causes userspace (e.g. echo) to retry the write infinitely.
🛡️
Validate all store() input
Use kstrtoint(), kstrtoul(). Always range-check. Never apply an out-of-range value — hardware can break or wedge.
🧹
Cleanup order matters
Remove debugfs → procfs → device → cdev → class → chrdev region. Reverse of creation order. Any deviation causes kernel oops on rmmod.
🔒
Lock shared state in show/store
sysfs callbacks run in process context and can be preempted. Always hold your mutex when reading/writing driver state to avoid races.
📁
Group your sysfs attrs
Use ATTRIBUTE_GROUPS() and device_create_with_groups() rather than calling device_create_file() for each one — atomic creation/removal, no partial state.

🎯 Decision Guide: Which Interface to Use?

1
Is this a stable user-visible attribute (a setting users will script)?
→ Use sysfs. Create a device_attribute. It becomes permanent ABI once merged upstream.
2
Is this a runtime statistics page with multiple values?
→ Use procfs. Write a seq_file-backed /proc entry. Perfect for status dumps, tables, multi-line stats.
3
Is this for debugging during development only?
→ Use debugfs. Register a hex dump, raw register access, or internal state. Remove before shipping or guard with #ifdef DEBUG.

Quiz Test your understanding

Q1 — Your sysfs store() callback successfully sets a value. What must you return?
AReturn 0 to signal success
BReturn count (number of bytes written)
CReturn 1 (true)
DReturn PAGE_SIZE
✓ Correct! Returning count is mandatory. Returning 0 tells userspace nothing was consumed, causing infinite retries.
✗ Wrong. You must return count on success. Returning 0 causes infinite loops in callers like echo.
Q2 — You're writing a /proc file that outputs a 100-line table. What's the correct implementation approach?
AImplement .read directly with copy_to_user()
Bsprintf() everything into a static buffer, return it once
CUse seq_file with single_open() + seq_printf()
DWrite it as a sysfs attribute instead
✓ Correct! seq_file handles partial reads, buffer growing, and cursor management automatically. Always use it for /proc.
✗ Wrong. seq_file is the correct approach. Raw read() implementations break with large output and partial reads.
Q3 — Which interface should expose internal register values for development debugging only?
Asysfs (it has the most features)
Bprocfs (it's always available)
Cdebugfs (no ABI contract, intended for debug)
Dioctl (it's more secure)
✓ Correct! debugfs has no ABI guarantee and is explicitly designed for this. sysfs and procfs entries become ABI once shipped.
✗ Wrong. debugfs is designed for this exact purpose — no ABI contract, freely changeable, intended only for debugging.
